## 💻 Ejemplo Guiado

### Docker Compose para Producción

```yaml
# docker-compose.prod.yml
version: "3.8"

services:
  # Frontend Astro
  frontend:
    image: ghcr.io/${{ github.repository }}-frontend:latest
    container_name: astro-frontend
    ports:
      - "4321:4321"
    depends_on:
      - backend
    networks:
      - prod-network
    restart: always

  # Backend FastAPI
  backend:
    image: ghcr.io/${{ github.repository }}-backend:latest
    container_name: fastapi-backend
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=${DATABASE_URL}
      - SECRET_KEY=${SECRET_KEY}
      - ENVIRONMENT=production
    depends_on:
      - db
    networks:
      - prod-network
    restart: always
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 3s
      retries: 3

  # PostgreSQL
  db:
    image: postgres:15
    container_name: postgres-prod
    environment:
      - POSTGRES_USER=${POSTGRES_USER}
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
      - POSTGRES_DB=${POSTGRES_DB}
    volumes:
      - prod-db-/var/lib/postgresql/data
    networks:
      - prod-network
    restart: always

  # n8n Automation
  n8n:
    image: n8nio/n8n:latest
    container_name: n8n-prod
    ports:
      - "5678:5678"
    environment:
      - N8N_HOST=${N8N_HOST}
      - WEBHOOK_URL=${WEBHOOK_URL}
    volumes:
      - n8n-/home/node/.n8n
    networks:
      - prod-network
    restart: always

volumes: prod-db-
  n8n-

networks:
  prod-network:
    driver: bridge
```


### Script de Deploy (deploy.sh)

```bash
#!/bin/bash
# deploy.sh

echo "🚀 Iniciando deploy a producción..."

# 1. Pull de últimas imágenes
docker-compose -f docker-compose.prod.yml pull

# 2. Detener servicios actuales
docker-compose -f docker-compose.prod.yml down

# 3. Iniciar nuevos servicios
docker-compose -f docker-compose.prod.yml up -d

# 4. Esperar que los servicios estén listos
sleep 10

# 5. Health check
echo "🔍 Verificando salud de servicios..."
curl -f http://localhost:8000/health || exit 1
curl -f http://localhost:4321/ || exit 1

# 6. Limpieza
docker system prune -f

echo "✅ Deploy completado exitosamente!"
```

### Monitoreo y Logs

```bash
# Ver logs de todos los servicios
docker-compose -f docker-compose.prod.yml logs -f

# Ver logs de un servicio específico
docker-compose -f docker-compose.prod.yml logs -f backend

# Ver estado de contenedores
docker-compose -f docker-compose.prod.yml ps

# Ver uso de recursos
docker stats

# Backup de base de datos
docker exec postgres-prod pg_dump -U user appdb > backup.sql
```

## ✅ Check de Comprensión

**1. ¿Qué comando reinicia todos los servicios?**

```markdown
- `docker-compose restart`
- `docker-compose down && docker-compose up -d` ✅
- `docker-compose refresh`
```

**2. ¿Por qué hacer health checks después del deploy?**

```markdown
- Verificar que los servicios están funcionando ✅
- Mejorar el rendimiento
- Limpiar caches
```

**3. ¿Dónde se guardan los backups de PostgreSQL?**

```markdown
- En el contenedor
- En volúmenes Docker persistentes ✅
- En la memoria RAM
```
