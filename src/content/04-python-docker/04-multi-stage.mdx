## 📚 Teoría Esencial

### ¿Qué es Multi-Stage Build?

| **Concepto**           | **Descripción**                          | **Beneficio** |
| ---------------------- | ---------------------------------------- | ------------- |
| **Build Stage**        | Etapa para compilar/construir            | Herramientas completas |
| **Production Stage**   | Etapa final para producción              | Solo lo necesario |
| **Copy Between**       | Copiar artifacts entre etapas            | Imagen limpia |
| **Size Reduction**     | Imagen final más pequeña                 | 70-90% reducción |

---

### Comparación de Tamaños de Imagen

| **Tipo**               | **Tamaño Aproximado**                    | **Uso** |
| ---------------------- | ---------------------------------------- | ------- |
| **Single-Stage**       | ~1GB (python:3.12 completo)              | Desarrollo |
| **Multi-Stage Slim**   | ~150MB (python:3.12-slim)                | Producción |
| **Multi-Stage Alpine** | ~50MB (python:3.12-alpine)               | Producción optimizada |

## 💻 Ejemplo Guiado

### Dockerfile Multi-Stage Básico

```dockerfile
# Dockerfile
# ============================================
# ETAPA 1: Build
# ============================================
FROM python:3.12-slim AS builder

WORKDIR /app

# Instalar dependencias de build
RUN apt-get update && apt-get install -y \
    gcc \
    && rm -rf /var/lib/apt/lists/*

# Copiar y instalar dependencias
COPY requirements.txt .
RUN pip install --no-cache-dir --user -r requirements.txt

# ============================================
# ETAPA 2: Production
# ============================================
FROM python:3.12-slim AS production

WORKDIR /app

# Copiar solo lo necesario desde builder
COPY --from=builder /root/.local /root/.local
COPY --from=builder /app .

# Configurar PATH
ENV PATH=/root/.local/bin:$PATH

# Usuario no root
RUN useradd -m -u 1000 appuser && chown -R appuser:appuser /app
USER appuser

EXPOSE 8000

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### Multi-Stage con Alpine (Mínimo)

```dockerfile
# Dockerfile
# ============================================
# ETAPA 1: Build
# ============================================
FROM python:3.12 AS builder

WORKDIR /app

# Instalar dependencias
COPY requirements.txt .
RUN pip install --no-cache-dir --user -r requirements.txt

# ============================================
# ETAPA 2: Production (Alpine)
# ============================================
FROM python:3.12-alpine AS production

WORKDIR /app

# Copiar dependencias instaladas
COPY --from=builder /root/.local /root/.local
COPY --from=builder /app .

ENV PATH=/root/.local/bin:$PATH

# Usuario no root
RUN adduser -D appuser && chown -R appuser:appuser /app
USER appuser

EXPOSE 8000

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### Comandos para Verificar Tamaño

```bash
# Construir imagen
docker build -t mi-app:latest .

# Ver tamaño de imagen
docker images mi-app

# Comparar tamaños
docker images | grep mi-app

# Ejecutar y verificar
docker run -d -p 8000:8000 --name mi-app mi-app:latest

# Ver logs
docker logs mi-app

# Limpiar
docker stop mi-app && docker rm mi-app
```

## ✅ Check de Comprensión

**1. ¿Qué keyword define una etapa en multi-stage build?**
````markdown
- `STAGE`
- `AS` ✅
- `PHASE`
````
**2. ¿Cómo se copian archivos entre etapas?**
````markdown
- `COPY --from=etapa` ✅
- `COPY --stage=etapa`
- `TRANSFER --from=etapa`
````
**3. ¿Qué ventaja tiene multi-stage build?**
````markdown
- Imágenes más pequeñas ✅
- Más rápido el build
- Mejor seguridad
````
