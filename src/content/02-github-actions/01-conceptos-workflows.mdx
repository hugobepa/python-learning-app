## 📚 Teoría Esencial

### ¿Qué es CI/CD?

| **Concepto**           | **Descripción**                          | **Beneficio** |
| ---------------------- | ---------------------------------------- | ------------- |
| **CI (Continuous Integration)** | Integrar código frecuentemente | Detecta errores temprano |
| **CD (Continuous Delivery)** | Entregar código listo para producción | Deploy rápido y seguro |
| **CD (Continuous Deployment)** | Deploy automático a producción | Sin intervención manual |

---

### Componentes de GitHub Actions

| **Componente**         | **Propósito**                          | **Ejemplo** |
| ---------------------- | -------------------------------------- | ----------- |
| **Workflow**           | Flujo de automatización completo       | `.github/workflows/ci.yml` |
| **Job**                | Conjunto de pasos que se ejecutan      | `build`, `test`, `deploy` |
| **Step**               | Acción individual dentro de un job     | `checkout`, `install`, `test` |
| **Action**             | Paso reutilizable predefinido          | `actions/checkout@v4` |
| **Runner**             | Servidor donde se ejecuta el workflow  | `ubuntu-latest` |

---

## 💻 Ejemplo Guiado

### Estructura de un Workflow

```yaml
# .github/workflows/ci.yml
name: CI Pipeline

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
      
      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      
      - name: Install dependencies
        run: pip install -r requirements.txt
      
      - name: Run tests
        run: pytest
```

### Trigger Events

```yaml
# Ejecutar en push a main
on:
  push:
    branches: [main]

# Ejecutar en cualquier branch
on: push

# Ejecutar en pull requests
on:
  pull_request:
    branches: [main]

# Ejecutar manualmente
on:
  workflow_dispatch:

# Ejecutar en schedule (cron)
on:
  schedule:
    - cron: '0 0 * * *'  # Daily at midnight
```

## 🎮 Ejercicio Práctico

**Reto:** Crea tu primer workflow
````markdown
1. En tu repositorio, crea la carpeta `.github/workflows/`
2. Crea un archivo `ci.yml`
3. Añade un workflow que:
   - Se ejecute en push a `main`
   - Use Python 3.12
   - Imprima "Hola GitHub Actions"
4. Haz commit y push
5. Verifica en la pestaña Actions de GitHub
````
```yaml
# Tu código aquí
# .github/workflows/ci.yml
name: Primer Workflow

on:
  push:
    branches: [main]

jobs:
  hello:
    runs-on: ubuntu-latest
    
    steps:
      - name: Saludar
        run: echo "Hola GitHub Actions"
      
      - name: Verificar Python
        run: python --version
```
