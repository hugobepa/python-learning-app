## 📚 Teoría Esencial

### Tipos de Configuración en GitHub Actions

| **Tipo**        | **Propósito**                         | **Visibilidad** | **Ejemplo**              |
| --------------- | ------------------------------------- | --------------- | ------------------------ |
| **Secrets**     | Datos sensibles (contraseñas, tokens) | Oculto en logs  | `API_KEY`, `DB_PASSWORD` |
| **Variables**   | Configuración no sensible             | Visible en logs | `ENVIRONMENT`, `VERSION` |
| **Environment** | Agrupación de secrets por entorno     | Por entorno     | `production`, `staging`  |

---

### Niveles de Secrets y Variables

| **Nivel**        | **Alcance**                         | **Uso**                   |
| ---------------- | ----------------------------------- | ------------------------- |
| **Repository**   | Todos los workflows del repositorio | `secrets.API_KEY`         |
| **Environment**  | Solo workflows de ese entorno       | `environments.production` |
| **Organization** | Todos los repositorios de la org    | `org.secrets.TOKEN`       |

## 💻 Ejemplo Guiado

### Configurar Secrets en GitHub

```bash
# 1. Ve a GitHub → Tu repositorio → Settings
# 2. Click en "Secrets and variables" → "Actions"
# 3. Click en "New repository secret"
# 4. Añade:
#    - Name: API_KEY
#    - Value: tu_clave_secreta_aqui
# 5. Click en "Add secret"
```

### Usar Secrets en Workflow

```yaml
# .github/workflows/deploy.yml
name: Deploy with Secrets

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Deploy with secrets
        env:
          API_KEY: ${{ secrets.API_KEY }}
          DB_PASSWORD: ${{ secrets.DB_PASSWORD }}
        run: |
          echo "Deploying with API key..."
          python deploy.py

      - name: Use variable
        env:
          ENVIRONMENT: ${{ vars.ENVIRONMENT }}
          VERSION: ${{ vars.VERSION }}
        run: |
          echo "Environment: $ENVIRONMENT"
          echo "Version: $VERSION"
```

### Secrets Protegidos por Entorno

```yaml
# .github/workflows/production.yml
name: Production Deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment: production # Requiere aprobación si está configurado

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Deploy to production
        env:
          PROD_API_KEY: ${{ secrets.PROD_API_KEY }}
          PROD_DB_URL: ${{ secrets.PROD_DB_URL }}
        run: |
          echo "Deploying to production..."
          python deploy_prod.py
```

## 🎮 Ejercicio Práctico

**Reto:** Configura secrets para tu proyecto

```markdown
1. Crea 2 secrets en tu repositorio:
   - `API_KEY` con un valor de prueba
   - `DEPLOY_TOKEN` con un token de prueba
2. Crea 2 variables:
   - `ENVIRONMENT` = `development`
   - `VERSION` = `1.0.0`
3. Crea un workflow que use los secrets y variables
4. Verifica que los secrets no aparecen en los logs
```

-- **Pista:** Usa el ejemplo guiado como referencia

```yaml
# Tu código aquí
# .github/workflows/secrets-test.yml
name: Secrets Test

on:
  workflow_dispatch:

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Use secrets
        env:
          API_KEY: ${{ secrets.API_KEY }}
          DEPLOY_TOKEN: ${{ secrets.DEPLOY_TOKEN }}
        run: |
          echo "API Key length: ${#API_KEY}"
          echo "Token configured: Yes"

      - name: Use variables
        env:
          ENVIRONMENT: ${{ vars.ENVIRONMENT }}
          VERSION: ${{ vars.VERSION }}
        run: |
          echo "Environment: $ENVIRONMENT"
          echo "Version: $VERSION"
```
